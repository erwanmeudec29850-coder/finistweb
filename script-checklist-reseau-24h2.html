Checklist mot de passe réseau — SMB 24H2

Toutes les étapes dans l'ordre pour régler une demande de mot de passe réseau inattendue sous Windows 11 24H2. À suivre pas à pas, sans rien oublier, sans mot de passe qui traîne en clair.

23 juin 2026checklist PDF

Si tu arrives ici depuis l'article, tu sais déjà ce qui se passe : Windows 11 24H2 a durci les règles de signature SMB, et un poste qui marchait sans problème redemande maintenant un mot de passe à chaque démarrage. Cette checklist rassemble les étapes dans l'ordre, prêtes à suivre directement sur le poste. Tu la télécharges une fois, tu l'as pour tous les tickets de ce type.

Ce que contient la checklist

  1. Vérifier la version Windows — confirmer qu'on est bien sur 24H2 avant de chercher ailleurs.
  2. Diagnostiquer avec PowerShell — la commande Get-SmbClientConfiguration pour voir si RequireSecuritySignature est activé.
  3. Vérifier la compatibilité côté serveur — NAS ou serveur de fichiers : est-ce qu'il supporte la signature SMB ?
  4. Enregistrer l'identifiant proprement — via le Gestionnaire d'identifiants Windows, en saisie interactive, sans mot de passe en clair dans aucune commande.
  5. Recréer le lecteur réseau persistent — mappage sans mot de passe dans la commande, en s'appuyant sur le credential déjà stocké.
  6. Vérifier au redémarrage — confirmer que le lecteur se monte automatiquement sans popup.
